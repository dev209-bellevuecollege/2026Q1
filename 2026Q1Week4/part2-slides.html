What is an API?

Application Programming Interface

The Waiter Analogy

An API is like a waiter at a restaurant:

  • You (client) make a request from the menu
  • Waiter (API) takes your order to the kitchen
  • Kitchen (server) prepares the food
  • Waiter brings back your meal (response)

Key Point: You don't need to know HOW the kitchen works. You just need to know the menu (API documentation).

Web APIs

A Web API allows different applications to communicate over HTTP.

Examples

Weather API: Get current weather data

Twitter API: Post tweets, read timelines

Stripe API: Process payments

Your Todo API: Manage todo items

REST

What is REST?

Representational State Transfer

REST Principles

REST is an architectural style for designing APIs. It's a set of conventions, not a strict protocol.

Key Characteristics

  • • Stateless: Each request is independent
  • • Resource-based: URLs represent "things"
  • • HTTP methods: Actions on resources
  • • Standard formats: Usually JSON

Resource-Based URLs

// Resources are NOUNS, not verbs

✓ /users

✓ /users/123

✓ /users/123/posts

// NOT like this:

✗ /getUser

✗ /deleteUser/123

The HTTP method defines the action, the URL defines the resource.

JSON: The Data Format

JavaScript Object Notation

JSON Syntax

{
  "id": 123,
  "title": "Buy groceries",
  "completed": false,
  "tags": ["shopping", "urgent"],
  "user": {
    "id": 1,
    "name": "Alice"
  }
}

JSON Rules

  • • Keys must be double-quoted strings
  • • Values: strings, numbers, booleans, null, arrays, objects
  • • No trailing commas
  • • No comments
  • • No undefined

JavaScript Conversion

JSON.parse(string) → Object

JSON.stringify(obj) → String

key

API Authentication

Proving Who You Are

Common Methods

API Keys

A secret key passed in header or query param

X-API-Key: your-api-key-here

Bearer Tokens (JWT)

Token received after login, sent with each request

Authorization: Bearer eyJhbGci...

Auth Flow (Your Todo App)

1. Register

POST /users → Create account

2. Login

POST /login → Get authToken

3. Use Token

Include in Authorization header

4. Access Protected Routes

GET /todos (with token)

Reading API Documentation

Swagger / OpenAPI

What is OpenAPI/Swagger?

A standard format for describing REST APIs. Swagger UI provides an interactive way to explore and test APIs.

What You'll Find

• Available endpoints (paths)

• HTTP methods for each endpoint

• Request parameters & body schema

• Response formats & status codes

• Authentication requirements
