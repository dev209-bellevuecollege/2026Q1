Why HTTPS?

The "S" Stands for Secure

HTTP Problems

Plain HTTP is sent in clear text. Anyone on the network can read your data!

Risks:

  • Passwords visible to attackers
  • Session hijacking
  • Data tampering
  • Man-in-the-middle attacks

HTTPS Benefits

HTTPS encrypts all communication between browser and server.

Provides:

  • Encryption: Data is scrambled
  • Authentication: Verify server identity
  • Integrity: Detect tampering

Look for the padlock icon in your browser's address bar!

How HTTPS Works

SSL/TLS Handshake (Simplified)

1. Hello
→
2. Certificate
→
3. Key Exchange
→
4. Encrypted!

The Process

1. Client Hello: Browser says "I want a secure connection"

2. Certificate: Server sends its SSL certificate (proof of identity)

3. Key Exchange: They agree on encryption keys

4. Encrypted: All data is now encrypted

SSL Certificates

A certificate proves the server is who it claims to be. Issued by trusted Certificate Authorities (CAs).

Certificate Warnings

Expired, self-signed, or mismatched certificates trigger browser warnings. Don't ignore them!

Collecting User Input

Basic Form Structure

  <label for="email">Email:</label>

    type="email"

    required

  <label for="password">Password:</label>
  <input
    type="password"
    id="password"
    name="password"
  >

  <button type="submit">Login</button>
</form>

Key Attributes

action - URL to send data to

method - HTTP method (GET or POST)

name - Key for the data (required!)

Default Behavior

Forms cause a full page reload when submitted. We usually want to prevent this!

JS

Handling Forms with JavaScript

preventDefault + fetch

Intercept the Submit

const form = document.querySelector('form');

form.addEventListener('submit', async (event) => {
  // Stop the page reload!
  event.preventDefault();

  // Get form data
  const formData = new FormData(form);
  const data = Object.fromEntries(formData);

  // Send via fetch instead
  const response = await fetch('/api/login', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify(data)
  });

  const result = await response.json();
  console.log(result);
});

Why This Pattern?

No page reload = Better UX

Custom headers = Send JSON, auth tokens

Handle response = Update UI, show errors

FormData API

new FormData(form) collects all input values by their name attribute.

Assignment 4 Example

Login Form Pattern

HTML

<form id="login-form">
  <input
    type="text"
    name="username"
    placeholder="Username"
    required
  >
  <input
    type="password"
    name="password"
    placeholder="Password"
    required
  >
  <button type="submit">Login</button>
</form>

JavaScript

const form = document.querySelector('#login-form');

form.addEventListener('submit', async (e) => {
  e.preventDefault();
  const data = Object.fromEntries(
    new FormData(form)
  );

  const res = await fetch('/login', {
    method: 'POST',
    headers: {'Content-Type': 'application/json'},
    body: JSON.stringify(data)
  });

  const { authToken } = await res.json();

  // Store token as cookie!
  document.cookie = `authToken=${authToken}`;
});

This pattern combines: Forms + fetch + Cookies. Use it for Assignment 4!
