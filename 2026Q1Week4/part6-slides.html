API Call Pattern

Using the authToken

After Login: Store Token

// Login response contains authToken
const { authToken } = await response.json();

// Store the token using appropriate storage
// (You decide: cookie, localStorage, or
//  sessionStorage?)

Think: What are the tradeoffs of each storage type for auth tokens?

Making Authenticated Requests

// Retrieve your stored token
const token = getStoredToken();

const response = await fetch(
  'http://localhost:3000/todos',
  {
    method: 'GET',
    headers: {
      'Authorization': `Bearer ${token}`
    }
  }
);

The Authorization: Bearer <token> header is required for all authenticated API calls!
