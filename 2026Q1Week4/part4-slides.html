Three Storage Options

Persisting Data in the Browser

Cookies

Small data sent to server with every request. Best for authentication.

localStorage

Persistent storage shared across all tabs. Data survives browser close.

sessionStorage

Per-tab storage. Data cleared when tab closes. Each tab is isolated.

local

localStorage

Persistent, Cross-Tab Storage

Basic API

// Store data
localStorage.setItem('theme', 'dark');

// Retrieve data
const theme = localStorage.getItem('theme');

// Remove item
localStorage.removeItem('theme');

// Clear everything
localStorage.clear();

Storing Objects

// Objects must be stringified!
const user = { name: 'Alice', age: 25 };

// Save
localStorage.setItem(
  'user',
  JSON.stringify(user)
);

// Load
const loaded = JSON.parse(
  localStorage.getItem('user')
);

Use Case: User preferences, theme settings, cached data, cross-tab shared state

session

sessionStorage

Per-Tab, Temporary Storage

Same API as localStorage

// Store data (this tab only)
sessionStorage.setItem(
  'gameState',
  JSON.stringify(currentGame)
);

// Retrieve data
const game = JSON.parse(
  sessionStorage.getItem('gameState')
);

// Remove / Clear
sessionStorage.removeItem('gameState');
sessionStorage.clear();

Key Difference

Each tab has its own sessionStorage. Opening a new tab = new, empty storage.

When to Use

  • • Form data that shouldn't persist
  • • Single-tab game state
  • • Temporary wizard/flow data
  • • Data that should reset per session

Key Point: Perfect for data that should reset when the user closes the tab.

cookie

Cookies

Server-Accessible Storage

Setting Cookies

// Basic cookie
document.cookie = "username=Alice";

// With expiration (seconds)
document.cookie = "token=abc123; max-age=3600";

// With path and secure
document.cookie = "authToken=xyz; path=/; secure";

// Delete (set max-age to 0)
document.cookie = "token=; max-age=0";

Reading Cookies

// Returns ALL cookies as one string
console.log(document.cookie);
// "username=Alice; token=abc123"

// Helper function to get one cookie
function getCookie(name) {
  const cookies = document.cookie.split('; ');
  for (const cookie of cookies) {
    const [key, val] = cookie.split('=');
    if (key === name) return val;
  }
  return null;
}

Use Case: Authentication tokens, session IDs, data the server needs to access

lock

Cookie Security Flags

Protecting Sensitive Data

Security Attributes

Secure

Only sent over HTTPS connections

cookie=value; Secure

HttpOnly

JavaScript cannot access (set by server only)

Set-Cookie: token=abc; HttpOnly

SameSite

Controls cross-site sending

Strict | Lax | None

SameSite Values

Strict: Never sent cross-site

Lax: Sent on top-level navigations (default)

None: Always sent (requires Secure)

Security Note

Auth tokens should ideally be HttpOnly (server-set). In learning environments, JS-accessible cookies are often used for simplicity.

Which Storage to Use?

Decision Framework

Ask These Questions

1. Does the server need this data?

Yes → Cookies

2. Should it persist after browser close?

Yes → localStorage or Cookies

3. Should each tab have its own copy?

Yes → sessionStorage

4. Should all tabs share this data?

Yes → localStorage

Common Use Cases

Shopping cart items:

→ localStorage (persist across sessions)

Form wizard progress:

→ sessionStorage (reset on new visit)

User login session:

→ Cookie (server needs it)
