What is fetch()?

The Modern Way to Make HTTP Requests

The Basics

fetch() is a built-in browser function that makes HTTP requests and returns a Promise.

// Simple GET request
fetch('https://api.example.com/users')
  .then(response => response.json())
  .then(data => {
    console.log(data);
  });

What's a Promise?

A Promise represents a value that will be available in the future (when the HTTP request completes).

Promise States

Pending: Request in progress
Fulfilled: Success! Data received
Rejected: Error occurred

.then() runs when the promise resolves. Chain them for multi-step operations.

async

Async/Await

Cleaner Promise Syntax

With .then() (Verbose)

fetch('https://api.example.com/users')
  .then(response => {
    return response.json();
  })
  .then(data => {
    console.log(data);
  })
  .catch(error => {
    console.error(error);
  });

With async/await (Clean!)

async function getUsers() {
  try {
    const response = await fetch(
      'https://api.example.com/users'
    );
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}

Key: await pauses execution until the Promise resolves. Only works inside async functions.

GET

GET Requests

Fetching Data

Basic GET

async function getTodos() {
  const response = await fetch(
    'https://jsonplaceholder.typicode.com/todos'
  );

  const todos = await response.json();

  console.log(todos);
  // Array of todo objects
}

GET with Auth Header

async function getMyTodos(token) {
  const response = await fetch(
    'https://api.example.com/todos',
    {
      headers: {
        'Authorization': `Bearer ${token}`
      }
    }
  );

  return response.json();
}

Remember: fetch() returns a Response object. Call .json() to parse the body!

POST

POST Requests

Sending Data

Creating a Resource

async function createTodo(token, todoData) {
  const response = await fetch(
    'https://api.example.com/todos',
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${token}`
      },
      body: JSON.stringify(todoData)
    }
  );

  return response.json();
}

Usage

const newTodo = {
  title: 'Learn fetch API',
  description: 'Master async requests',
  completed: false
};

const created = await createTodo(
  myToken,
  newTodo
);

console.log(created.id); // New ID

Key Points

• Set method: 'POST'
• Set Content-Type header
• JSON.stringify() the body

!

Error Handling

Two Types of Errors

Network Errors

Caught by try/catch: No internet, server down, DNS failure, CORS blocked

HTTP Errors

NOT caught by try/catch: 404, 401, 500 responses. fetch() considers these "successful" requests!

Proper Error Handling

async function fetchData(url) {
  try {
    const response = await fetch(url);

    // Check for HTTP errors!
    if (!response.ok) {
      throw new Error(
        `HTTP error: ${response.status}`
      );
    }

    return await response.json();
  } catch (error) {
    console.error('Fetch failed:', error);
    throw error;
  }
}

Always check response.ok or response.status before processing!

SOP

Same-Origin Policy

Understanding Browser Security

What is an "Origin"?

Origin = Protocol + Domain + Port

✓ https://example.com:443

✓ https://example.com (same)

✗ http://example.com (protocol)

✗ https://api.example.com (domain)

✗ https://example.com:8080 (port)

Why Same-Origin Policy?

Security Threat Example

Without SOP, a malicious site could:

1. Load in an iframe while you're logged in to your bank

2. Use JavaScript to read your account data

3. Transfer money without your knowledge

SOP prevents this by blocking scripts from accessing data from different origins.

Same-Origin Policy is a browser security feature that isolates origins from each other.

CORS

CORS Explained

Cross-Origin Resource Sharing

The Problem

Access to fetch at 'https://api.com' from origin 'https://mysite.com' has been blocked by CORS policy

Why? Your app at mysite.com is trying to fetch from a different origin api.com.

The Solution

The server must include CORS headers to explicitly allow cross-origin requests.

Access-Control-Allow-Origin: *

or specific origin:

Access-Control-Allow-Origin: https://mysite.com

Key Point: CORS is a server-side configuration. You cannot fix CORS errors from your JavaScript code!

OPTIONS

Preflight Requests

When Browsers Ask Permission First

Simple vs Preflighted

Simple Requests (No Preflight)

• GET, HEAD, or POST only
• Only standard headers
• Content-Type: text/plain, form-data, or urlencoded

Preflighted Requests

• PUT, DELETE, PATCH methods
• Custom headers (Authorization)
• Content-Type: application/json

What Happens?

// Your code
fetch('https://api.example.com/todos', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer token'
  }
});

// Browser automatically sends OPTIONS first:
OPTIONS /todos
// Server responds with allowed methods/headers
// Then browser sends actual POST request

Preflight is automatic. You'll see OPTIONS requests in DevTools Network tab.

Debug

Debugging CORS

Common Issues and Solutions

Use DevTools Network Tab

1. Open DevTools (F12)
2. Go to Network tab
3. Look for the failed request (red)
4. Check Response Headers for CORS headers

Common Causes

• Server not sending CORS headers
• Wrong origin in Allow-Origin
• Missing credentials handling
• Preflight failing (OPTIONS denied)

What You Can Control

Solutions

✓ Use APIs that support CORS
✓ Use your own backend as proxy
✓ During development: browser extensions
✓ Configure server (if you control it)
