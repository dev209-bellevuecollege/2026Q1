What is HTTP?

HyperText Transfer Protocol

The Web's Language

HTTP is the protocol (set of rules) that browsers and servers use to communicate.

Key Characteristics

  • • Human-readable (easy to debug)
  • • Stateless (no memory between requests)
  • • Extensible (via headers)
  • • Client-server model

Client-Server Model

Browser (Client)
Request ↓
Internet
↓ Response
Web Server

The client always initiates the conversation. The server only responds.

⇄

The Request/Response Cycle

A Conversation in Two Parts

HTTP Request

GET /users/123 HTTP/1.1
Host: api.example.com
Accept: application/json
Authorization: Bearer abc123

Method: What action to take

Path: What resource to access

Headers: Extra information

Body: Data to send (optional)

HTTP Response

HTTP/1.1 200 OK
Content-Type: application/json
Content-Length: 82

{
  "id": 123,
  "name": "Alice",
  "email": "alice@example.com"
}

Demo: Open DevTools (F12) → Network tab → Refresh page → Click any request

CRUD

HTTP Methods

The Verbs of the Web

Method Purpose Has Body? Example
GET Retrieve data No Get user profile
POST Create new resource Yes Create new user
PUT Replace entire resource Yes Update user completely
PATCH Partial update Yes Update just email
DELETE Remove resource No Delete user

CRUD Mapping: Create=POST, Read=GET, Update=PUT/PATCH, Delete=DELETE

200

Status Codes

The Server's Response

2xx Success

  • 200 OK
  • 201 Created
  • 204 No Content

3xx Redirect

  • 301 Moved Permanently
  • 302 Found (Temporary)
  • 304 Not Modified

4xx Client Error

  • 400 Bad Request
  • 401 Unauthorized
  • 403 Forbidden
  • 404 Not Found

5xx Server Error

  • 500 Internal Server Error
  • 502 Bad Gateway
  • 503 Service Unavailable

Quick Memory Trick

2xx = "Here you go!"
3xx = "Go look elsewhere"
4xx = "You messed up"
5xx = "I messed up"

HTTP is Stateless

But Sessions Exist?

The Problem

HTTP has no memory between requests. Each request is completely independent.

Request 1: "I'm Alice, log me in"
↓
Request 2: "Show my profile"
Server: "Who are you?" 🤔

The Solution: Cookies

Cookies allow the browser to remember information between requests.

Login Response: Set-Cookie: session=abc123
↓
Next Request: Cookie: session=abc123
Server: "Welcome back, Alice!"

We'll cover cookies, localStorage, and sessionStorage in detail in Part 4!
