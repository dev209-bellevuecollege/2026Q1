The JavaScript Problem

Why Can't JS Run Everywhere?

JavaScript Was Browser-Only

Originally, JavaScript could only run inside web browsers.

The Limitation

  • • Browsers have a JS engine built-in
  • • Chrome uses V8, Firefox uses SpiderMonkey
  • • No browser = No JavaScript execution
  • • Couldn't use JS for server-side code

What is a Runtime Engine?

A runtime engine is the program that reads and executes JavaScript code.

Think of it like:

Python needs the Python interpreter
Java needs the JVM (Java Virtual Machine)
JavaScript needs... a JS engine!

What if we could take the browser's JS engine and run it outside the browser?

Node

Enter Node.js

JavaScript Everywhere

What is Node.js?

Node.js is a JavaScript runtime built on Chrome's V8 engine that runs outside the browser.

Created in 2009

Ryan Dahl extracted Chrome's V8 engine and wrapped it with APIs for file system, networking, etc.

How It Works

VS

Browser JS vs Node.js

Same Language, Different Environments

Feature Browser JavaScript Node.js
DOM Access ✓ document, window ✗ No DOM
File System ✗ Security sandbox ✓ fs module
Network Servers ✗ Client only ✓ http module
User Interaction ✓ Events, UI ✗ No UI (command line)
Package Manager Via script tags or bundlers ✓ npm built-in

Same JavaScript syntax, but different global objects and APIs available!

Why Do We Need Node for React?

The Development Toolchain

React Runs in the Browser...

Your final React app runs in the browser as regular JavaScript.

...But Development Needs Node!

  • • npm - Package manager (install libraries)
  • • Vite - Development server
  • • Babel - Transpiles JSX to JavaScript
  • • ESLint - Code quality checking
  • • Build tools - Bundle for production

The Development Flow

You write JSX code
↓
Node.js runs build tools
↓
Transforms to browser-ready JS
↓
Browser runs the final code
npm

npm: Node Package Manager

The World's Largest Software Registry

What is npm?

npm comes bundled with Node.js and provides:

  • • Registry - 2M+ packages to download
  • • CLI tool - Install, update, manage packages
  • • Scripts - Run project commands

package.json

Every Node project has this file listing:

• Project name and version

• Dependencies (libraries used)

• Scripts (commands to run)

Common npm Commands

# Install all dependencies
npm install

# Add a new package
npm install react

# Run a script from package.json
npm run dev

# Create a new project
npm create vite@latest my-app

Run node --version and npm --version to verify installation!
