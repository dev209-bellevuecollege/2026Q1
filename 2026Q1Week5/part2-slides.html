Vite

Vite: The Modern Build Tool

Fast Development Server & Bundler

What is Vite?

Vite (French for "fast") is a build tool that provides:

  • • Instant dev server startup
  • • Hot Module Replacement (HMR)
  • • Optimized production builds

Why Not Create React App?

CRA is deprecated. Vite is the modern standard - much faster and more flexible.

Create a New React Project

# Create new project
npm create vite@latest my-app -- --template react

# Navigate into folder
cd my-app

# Install dependencies
npm install

# Start development server
npm run dev

Dev server runs at http://localhost:5173

React Project Structure

Understanding the File Layout

Directory Layout

my-app/ |-- index.html # Entry HTML file |-- package.json # Project config |-- vite.config.js # Vite settings |-- node_modules/ # Installed packages |-- public/ # Static assets | +-- vite.svg +-- src/ # Your code lives here! |-- main.jsx # App entry point |-- App.jsx # Root component |-- App.css # App styles +-- index.css # Global styles

Key Files Explained

index.html

Contains a single <div id="root"> where React injects all content.

src/main.jsx

Creates the React root and renders the App component into the DOM.

src/App.jsx

Your main component - the starting point of your UI.

.jsx Extension

Files with JSX syntax use .jsx (or .tsx for TypeScript)

HTML

The Entry Point: index.html

Where React Takes Over

index.html Contents

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport"
        content="width=device-width" />
  <title>Vite + React</title>
</head>
<body>
  <div id="root"></div>
  <script type="module"
          src="/src/main.jsx"></script>
</body>
</html>

What's Happening?

The Empty Container

<div id="root"> is initially empty. React fills it with content!

Module Script

type="module" enables ES6 imports. Vite processes main.jsx automatically.

Minimal HTML

The HTML stays simple - all UI is generated by JavaScript (React)!

main

The Bootstrap: main.jsx

Starting the React Application

main.jsx Contents

import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.jsx'
import './index.css'

ReactDOM.createRoot(
  document.getElementById('root')
).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
)

Line by Line

Lines 1-4: Import React, ReactDOM, your App component, and global CSS

createRoot(): Creates a React "root" attached to the #root div

render(): Tells React what to display in that root

StrictMode: Development helper that warns about potential problems (doesn't affect production)

This file rarely changes - it just bootstraps your app!

Development Workflow

Hot Module Replacement

npm Scripts (package.json)

"scripts": {
  "dev": "vite",
  "build": "vite build",
  "preview": "vite preview"
}

npm run dev - Start development

npm run build - Create production build

npm run preview - Preview production

Hot Module Replacement (HMR)

Instant updates! When you save a file, only that component updates - no full page reload.

The Dev Experience

  • 1. Start server with npm run dev
  • 2. Open http://localhost:5173
  • 3. Edit code, save file
  • 4. See changes instantly!
