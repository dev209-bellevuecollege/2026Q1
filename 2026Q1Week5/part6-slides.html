The Browser Problem

Browsers Don't Understand JSX!

What You Write

// Your code (JSX)
function App() {
  return (
    <div className="app">
      <h1>Hello World</h1>
    </div>
  );
}

Problem: Browsers can't run JSX. They only understand standard JavaScript!

What Browser Needs

// Standard JavaScript
function App() {
  return React.createElement(
    'div',
    { className: 'app' },
    React.createElement(
      'h1',
      null,
      'Hello World'
    )
  );
}

Solution: Transform JSX into regular JavaScript before sending to browser.

⇄

What is Transpilation?

Code-to-Code Transformation

Transpile vs Compile

Compile

High-level code → Machine code (C++ to binary)

Transpile

High-level code → High-level code (JSX to JS)

What Gets Transpiled?

JSX → React.createElement()

ES6+ imports → Browser-compatible format

Modern JS features → Older syntax for compatibility

TypeScript → JavaScript (if using TS)

Babel is the tool that transpiles your code. Vite handles this automatically!

dev

Development Mode

npm run dev

What Happens

1. Vite starts dev server
↓
2. Browser requests files
↓
3. Vite transpiles on-demand
↓
4. Browser receives JS + HMR

Key Features

Instant Server Start

No bundling needed - files served individually

Hot Module Replacement

Changes appear instantly without refresh

Source Maps

Debug original JSX, not transpiled code

NOT Optimized

Many small files, debug info included

prod

Production Build

npm run build

What Happens

1. Transpile all JSX to JS
↓
2. Bundle all JS into few files
↓
3. Minify (remove whitespace, shorten names)
↓
4. Output to /dist folder

Output: /dist Folder

dist/ |-- index.html +-- assets/ |-- index-a1b2c3d4.js # All JS bundled |-- index-e5f6g7h8.css # All CSS bundled +-- logo-i9j0k1l2.svg # Hashed assets

Production Optimizations

  • • Minified code (smaller file size)
  • • Tree shaking (removes unused code)
  • • Content hashes (cache busting)
VS

Development vs Production

Key Differences

Feature Development Production
Command npm run dev npm run build
Bundling No bundling (ES modules) All code bundled
Minification No (readable code) Yes (compressed)
Source Maps Yes (for debugging) Optional
Error Messages Detailed Minimal
File Size Large (many files) Small (optimized)

Remember: npm run build → upload /dist folder to web server!
