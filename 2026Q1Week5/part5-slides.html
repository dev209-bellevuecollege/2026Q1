What are Props?

Properties Passed to Components

Props = Function Arguments

Props are how you pass data from a parent to a child component.

// Parent passes props
<Greeting name="Alice" age={25} />

// Like calling a function:
Greeting({ name: "Alice", age: 25 })

Receiving Props

// Child receives props object
function Greeting(props) {
  return (
    <div>
      <h1>Hello, {props.name}!</h1>
      <p>Age: {props.age}</p>
    </div>
  );
}

props is always an object containing all passed attributes.

{ }

Destructuring Props

Cleaner Syntax

Without Destructuring

function UserCard(props) {
  return (
    <div>
      <h2>{props.name}</h2>
      <p>{props.email}</p>
      <p>{props.role}</p>
    </div>
  );
}

Lots of props. repetition!

With Destructuring

function UserCard({ name, email, role }) {
  return (
    <div>
      <h2>{name}</h2>
      <p>{email}</p>
      <p>{role}</p>
    </div>
  );
}

Cleaner! Shows exactly which props are expected.

Best Practice: Always destructure props in the function parameter!

Props Can Be Anything

Any JavaScript Value

Passing Different Types

<ProductCard
  // String
  title="MacBook Pro"

  // Number (use curly braces!)
  price={1999}

  // Boolean
  inStock={true}

  // Array
  tags={['laptop', 'apple']}

  // Object
  specs={{ ram: 16, storage: 512 }}

  // Function
  onBuy={handlePurchase}
/>

Syntax Rules

Strings

Use quotes: name="Alice"

Everything Else

Use curly braces: count={42}

Boolean Shorthand

disabled = disabled={true}

Double Braces for Objects

style={{ color: 'red' }}
Outer = JSX, Inner = object literal

key

The key Prop

Essential for Lists

Without key (Warning!)

const todos = [
  { id: 1, text: 'Learn React' },
  { id: 2, text: 'Build app' },
];

// This causes a warning!
{todos.map((todo) => (
  <li>{todo.text}</li>
))}

Warning: Each child in a list should have a unique "key" prop.

With key (Correct!)

// Add unique key prop
{todos.map((todo) => (
  <li key={todo.id}>
    {todo.text}
  </li>
))}

Why key Matters

  • • Helps React track which items changed
  • • Enables efficient DOM updates
  • • Prevents bugs with list reordering

Rule: Use unique, stable IDs as keys. Avoid using array index as key!

🔒

Props are Read-Only

One-Way Data Flow

Never Modify Props!

function BadComponent({ count }) {
  // NEVER do this!
  count = count + 1;  // Error!

  return <p>{count}</p>;
}

Props should be treated as immutable. Components must be "pure" with respect to their props.

Data Flows Down

Parent (owns the data)
↓ props
Child (receives data)
↓ props
Grandchild (receives data)

This is called unidirectional data flow - data only flows downward through props.

ReactDOM & Rendering

How React Updates the Page

The Root

import ReactDOM from 'react-dom/client';
import App from './App';

// 1. Find the DOM element
const container = document
  .getElementById('root');

// 2. Create a React root
const root = ReactDOM
  .createRoot(container);

// 3. Render the app
root.render(<App />);

How Rendering Works

1. Initial Render

React creates all components and builds the DOM

2. Re-render (when data changes)

React compares old vs new, updates only what changed

StrictMode

Renders components twice in dev to catch bugs (normal!)

React's virtual DOM makes updates efficient - only changed elements are modified!
