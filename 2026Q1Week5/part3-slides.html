What is React?

A JavaScript Library for Building UIs

React at a Glance

React is a JavaScript library developed by Facebook (Meta) for building user interfaces.

Core Philosophy

  • • Components: Break UI into reusable pieces
  • • Declarative: Describe what you want, not how
  • • Data-driven: UI updates when data changes

Why React?

Component Abstraction
Build complex UIs from simple, isolated pieces

Efficient Updates
React optimizes DOM changes automatically

Massive Ecosystem
Most popular framework; huge community

Industry Standard
Used by Netflix, Airbnb, Instagram, etc.

Thinking in Components

Breaking Down the UI

Traditional Approach

One big HTML file with JavaScript manipulating it

  • • Hard to maintain large pages
  • • Difficult to reuse code
  • • Mixing concerns

React Approach

UI as a tree of independent components

  • • Each component = one concern
  • • Easy to reuse and test
  • • Clear data flow

JSX

What is JSX?

JavaScript + XML

JSX Looks Like HTML in JS

// This is JSX!
const element = (
  <div className="card">
    <h1>Hello World</h1>
    <p>Welcome to React</p>
  </div>
);

Note: This is NOT HTML! It's JavaScript that looks like HTML.

What JSX Compiles To

// Babel transforms JSX to this:
const element = React.createElement(
  'div',
  { className: 'card' },
  React.createElement(
    'h1', null, 'Hello World'
  ),
  React.createElement(
    'p', null, 'Welcome to React'
  )
);

JSX is just syntactic sugar - makes code easier to write!

Rules

JSX Rules & Syntax

Important Differences from HTML

className, not class

// HTML:
<div class="box">

// JSX:
<div className="box">

class is reserved in JS

Self-Closing Tags

// HTML:
<img src="...">
<br>

// JSX:
<img src="..." />
<br />

Must close all tags

camelCase Attributes

// HTML:
onclick="..."
tabindex="1"

// JSX:
onClick={...}
tabIndex={1}

JS naming convention

Must Return Single Parent Element

// Error!
return (
  <h1>Title</h1>
  <p>Text</p>
);
// Correct!
return (
  <div>
    <h1>Title</h1>
    <p>Text</p>
  </div>
);

JavaScript in JSX

Embedding Expressions with Curly Braces

Curly Braces = JavaScript

const name = "Alice";
const age = 25;
const imageUrl = "photo.jpg";

const element = (
  <div>
    <h1>Hello, {name}</h1>
    <p>Age: {age} years</p>
    <p>Next year: {age + 1}</p>
    <img src={imageUrl} />
  </div>
);

What Can Go in { }?

✓ Yes!

  • • Variables: {name}
  • • Expressions: {a + b}
  • • Function calls: {getName()}
  • • Ternaries: {x ? 'a' : 'b'}

✗ No!

  • • Statements: {if(x)...}
  • • Loops: {for...}
  • • Objects directly: {{a:1}}

Think: "Would this work in console.log()?"

Rendering Arrays

Lists and map()

Arrays Render as Siblings

const fruits = ['Apple', 'Banana', 'Cherry'];

const list = (
  <ul>
    {fruits.map((fruit) => (
      <li>{fruit}</li>
    ))}
  </ul>
);

// Renders:
// <ul>
//   <li>Apple</li>
//   <li>Banana</li>
//   <li>Cherry</li>
// </ul>

The Pattern

array.map() returns a new array of JSX elements
