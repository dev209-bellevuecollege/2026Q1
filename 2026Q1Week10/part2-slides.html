Speaker Notes: Walk through each generation. Gen 1 (2023): Chat interfaces. You ask ChatGPT or Claude a question, copy the code, paste it into your editor. Gen 2 (2023-2024): Autocomplete tools like Copilot that suggest code inline as you type. Gen 3 (2024-2025): Agentic tools that understand your whole codebase, edit multiple files, run tests, and iterate. Emphasize that Gen 1 is still useful! Sometimes a simple chat question is the right tool.

Terminal-Based Agents

The "Big Three" CLI Tools

Claude Code

Anthropic

Agentic coding in your terminal. Reads your entire codebase, edits files, runs tests, creates commits and PRs.

  • • Multi-agent orchestration
  • • Voice mode (talk to it!)
  • • MCP extensibility
  • • Full git workflow

Codex CLI

OpenAI

Open-source, Rust-based terminal agent. Uses GPT-5-Codex for coding tasks. Multi-agent support with isolated git worktrees.

  • • Open source (Rust)
  • • GPT-5-Codex model
  • • Built-in code review
  • • Adjustable reasoning

Gemini CLI

Google

Free access to Gemini 2.5 Pro with your Google account. 60 requests/min, 1,000/day. Open source.

  • • Free tier (generous!)
  • • Google Search grounding
  • • Open source
  • • MCP extensibility

All three share a pattern: read your code → plan changes → edit files → run commands → iterate

Speaker Notes: These three are converging on similar capabilities. The main differences are the underlying model (Claude vs GPT-5 vs Gemini), ecosystem integrations, and pricing. Claude Code has the most mature multi-agent story. Codex CLI is open source and built in Rust. Gemini CLI has the most generous free tier. For students, Gemini CLI is the easiest to get started with since it's free. We'll demo Claude Code later. Point out that these all run in the terminal — no IDE required. You can use them with any editor.
IDE

AI-Native IDEs

The Editor IS the AI

Cursor

Anysphere

VS Code fork with AI as a core feature. Composer mode plans and executes multi-file edits. Visual editor for drag-and-drop UI changes.

  • • "Composer" agent for multi-file edits
  • • Visual editor: point-and-prompt UI
  • • Choose any model (Claude, GPT, Gemini)
  • • Most popular AI IDE (>2M users)

Windsurf

Cognition AI

"Cascade" agentic system handles multi-file edits autonomously. Free tier available.

  • • Cascade: autonomous multi-file agent
  • • Multi-model support
  • • Free tier to get started

GitHub Copilot

Microsoft

The original AI coding assistant. 20M+ users. Now has an autonomous coding agent that creates PRs from issues.

  • • Most widely adopted (90% Fortune 100)
  • • Autonomous PR creation from issues
  • • Works in VS Code, JetBrains, Neovim
  • • $10/month (Pro)

Key Difference from Terminal Agents

IDE tools give you a visual, side-by-side experience. You see the diffs in real time and accept/reject changes inline. Terminal agents give you more flexibility and can be used with ANY editor.

Speaker Notes: The difference between terminal agents and AI IDEs is mainly the interface. Terminal agents are editor-agnostic — you can use them with Vim, VS Code, or anything. AI IDEs give you a tighter feedback loop with inline diffs. Many developers use both: an AI IDE for day-to-day work and a terminal agent for bigger tasks. GitHub Copilot is the most widely adopted because it was first and has the biggest distribution through VS Code. Cursor is the most popular dedicated AI IDE. Point out that multi-model support is now standard — you're not locked in.
SPEC

Spec-Driven & Agent-First Platforms

Requirements Before Code

Amazon Kiro

AWS

Spec-driven IDE. Before writing code, Kiro generates:

  1. Requirements document (EARS notation)
  2. System design with architecture decisions
  3. Dependency-ordered task list

Only THEN does it write implementation code.

Google Antigravity

Google

Agent-first development platform with two modes:

  • Editor View: Traditional AI-powered IDE for synchronous coding
  • Manager View: Orchestrate multiple agents working in parallel across workspaces

Agents produce "artifacts" (plans, screenshots, recordings) for verification.

The philosophical difference: think first, code second. These tools formalize the planning step that good developers already do.

Speaker Notes: This is the most interesting category. Kiro's approach is "specs before code" — it forces you to think through requirements and design before writing implementation. This mirrors what experienced software engineers already do. Antigravity takes a different angle — the Manager View lets you delegate tasks to multiple AI agents that work simultaneously. Think of it as being the project manager of a team of AI developers. We'll demo Antigravity later. Emphasize that the spec-driven approach is particularly valuable for larger projects where diving straight into code leads to rework.
VIBE

"Vibe Coding" & Prompt-to-App

Describe an App, Get a Working App

What is "Vibe Coding"?

Term coined by Andrej Karpathy (OpenAI co-founder). Collins Dictionary's 2025 Word of the Year. It means using AI to generate code by describing what you want — accepting the output without deeply reviewing the structure.

v0 (Vercel)

Generates Next.js apps with beautiful UI. Best for design-focused workflows.

Bolt.new (StackBlitz)

Most framework flexibility. Full deployment pipelines and team workspaces.

Lovable

Produces clean React code. Most accessible for non-technical users. $6.6B valuation.

Power

Rapid prototyping, idea validation, accessibility for non-developers. Great for MVPs and hackathons.

Risk

Technical debt, security vulnerabilities, lack of understanding. Not suitable for production without review.

Speaker Notes: "Vibe coding" is a real cultural phenomenon. Y Combinator reported 25% of its Winter 2025 batch had codebases that were 95% AI-generated. The tools are genuinely impressive for prototyping. But there's a real tension: vibe coding is fast but fragile. For learning, it can be counterproductive if you never understand what was generated. For production, it creates technical debt. Discuss: "When is vibe coding appropriate?" Answer: prototypes, hackathons, idea validation. When is it NOT? Production code, security-sensitive features, code you need to maintain long-term.
