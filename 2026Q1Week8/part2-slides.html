What is DNS?

Domain Name System

The Problem

Computers find each other using IP addresses — numbers like 142.250.80.46

But humans can't remember numbers for every website they visit.

The Solution: DNS

DNS translates human-readable domain names into IP addresses.

google.com → 142.250.80.46

How DNS Lookup Works

You type google.com in browser
↓
Browser asks DNS resolver for IP
↓
Resolver queries root → TLD → authoritative servers
↓
IP address 142.250.80.46 returned
↓
Browser connects to that IP
REC

DNS Record Types

The Records You Need to Know

A Record

Maps a domain to an IPv4 address

example.com  →  76.76.21.21

Used for root/apex domains (no subdomain prefix).

CNAME Record

Maps a domain to another domain name

www.example.com  →  cname.vercel-dns.com

Used for subdomains (www, app, api, etc.).

Important Rule

You cannot use a CNAME on a root domain!

example.com → CNAME is not allowed by the DNS spec. Only subdomains like www.example.com can use CNAME.

The Workaround

ALIAS / ANAME records

Not part of the official DNS spec, but most registrars (Cloudflare, Namecheap) support them. They act like a CNAME but work on root domains.

Demo: Run nslookup google.com in your terminal to see a DNS lookup in action.

Custom Domains on Hosting Platforms

Connecting Your Domain to Your App

The Setup Workflow

  1. 1. Buy a domain from a registrar (Namecheap, Cloudflare, etc.)
  2. 2. Add the domain in your hosting platform's dashboard
  3. 3. Configure DNS records at your registrar
  4. 4. Wait for DNS propagation (minutes to hours)
  5. 5. Platform auto-provisions HTTPS certificate

Example: Vercel + Custom Domain

For root domain:

A Record: example.com → 76.76.21.21

For www subdomain:

CNAME: www → cname.vercel-dns.com

For Final Projects

The default *.vercel.app or *.netlify.app URLs are perfectly fine — no need to buy a domain!
