Vercel

The Recommended Frontend Platform

What is Vercel?

Built by the creators of Next.js. Optimized for frontend frameworks including React + Vite.

Deploy Workflow

  1. 1. Push React project to GitHub
  2. 2. Go to vercel.com → Import Project
  3. 3. Select your repo
  4. 4. Vercel auto-detects Vite, runs build
  5. 5. Live URL in ~60 seconds

Free Tier (Hobby Plan)

  • ✓ 100 GB bandwidth/month
  • ✓ 6,000 build minutes/month
  • ✓ Serverless functions included
  • ✓ Auto HTTPS on all deployments
  • ✓ No credit card required

GitHub Integration

  • • Every git push triggers auto-deploy
  • • PRs get preview deployment URLs
  • • Merge to main = production deploy

Vercel Serverless Functions

Backend API Without a Server

How It Works

Create files in an /api directory. Each file becomes an HTTP endpoint automatically.

Example: /api/hello.js

export default function handler(req, res) {
  res.status(200).json({
    message: 'Hello from API!',
    time: new Date().toISOString()
  });
}

Accessible at: yoursite.vercel.app/api/hello

When to Use This

Good For

  • ✓ Simple CRUD API endpoints
  • ✓ Form handling
  • ✓ Proxying third-party APIs
  • ✓ Hiding API keys from the frontend

Limitations

  • ✗ 10-second timeout on free tier
  • ✗ No WebSocket support
  • ✗ Stateless between requests
  • ✗ Not a full Express server
N

Netlify

Vercel's Closest Competitor

Key Features

  • ✓ Same GitHub auto-deploy workflow as Vercel
  • ✓ Native SPA support (client-side routing works)
  • ✓ Drag-and-drop deployment option
  • ✓ Netlify Functions (serverless, like Vercel)
  • ✓ Auto HTTPS on all deployments

Free Tier

  • • 100 GB bandwidth/month
  • • 300 build minutes/month
  • • Serverless functions included
  • • No credit card required

Netlify Functions

Place function files in netlify/functions/ directory. AWS Lambda under the hood.

Example: netlify/functions/hello.js

export const handler = async (event) => {
  return {
    statusCode: 200,
    body: JSON.stringify({
      message: "Hello from Netlify!"
    })
  };
};

Vercel vs. Netlify

Both are excellent. Pick whichever you prefer — the experience is nearly identical.

R

Render.com

When You Need a Real Backend Server

Why Render?

Unlike Vercel/Netlify, Render hosts actual persistent servers — not just serverless functions.

Good For

  • ✓ Full Express.js APIs
  • ✓ WebSocket connections
  • ✓ Background jobs
  • ✓ Traditional server architecture

Free Tier

  • • Free web services (Node.js, Python, etc.)
  • • Free static site hosting
  • • Free PostgreSQL database (90-day limit)
  • • GitHub auto-deploy
  • • No credit card required

The Free Tier Caveat

Services Sleep After 15 Minutes

Free-tier services spin down after 15 minutes of no traffic. The first request after sleeping takes 30–60 seconds to cold-start.

Workaround: Use a free cron service to ping your API every 10 minutes to keep it awake.

Deploy Workflow

  1. 1. Push Express project to GitHub
  2. 2. Go to render.com → New Web Service
  3. 3. Connect your GitHub repo
  4. 4. Set build command: npm install
  5. 5. Set start command: node server.js
  6. 6. Choose Free tier → Deploy

Other Options & What to Avoid

The Full Landscape

Worth Knowing About

Cloudflare Pages

Most generous free tier — unlimited bandwidth and requests. Great for static sites. Workers for serverless APIs. Slightly steeper learning curve.

GitHub Pages

Free for public repos. Good for simple static sites. SPA routing is broken — requires HashRouter or 404.html workaround. No backend support at all.

Avoid for Student Projects

Railway

Removed meaningful free tier. Only $1/month in credits after a 30-day trial. Requires payment for real use.

Fly.io

Requires a credit card to sign up. No meaningful free tier for new users. Steeper learning curve (Docker/containers).

Both Railway and Fly.io are excellent platforms — just not free enough for coursework.
