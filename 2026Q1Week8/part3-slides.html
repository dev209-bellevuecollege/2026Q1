Why This Separation?

CDN vs. Server — Different Jobs, Different Hosting

Characteristic Frontend (CDN) Backend API (Server)
What it is Static files from npm run build Running Node.js/Express process
Where it runs CDN edge nodes worldwide Single server or region
How it scales Automatically via CDN caching Requires scaling configuration
Cost at scale Very cheap (just bandwidth) More expensive (compute time)
Execution Code runs in user's browser Code runs on the server

A React app after npm run build is just static files — it doesn't need a server to "run." The browser downloads them and executes React client-side.

API

Two Backend Models

Persistent Server vs. Serverless Functions

Persistent Server

A Node.js/Express process that is always running, waiting for requests.

Pros

  • ✓ WebSocket support (real-time)
  • ✓ In-memory state between requests
  • ✓ Familiar Express.js code

Cons

  • ✗ Costs more to run 24/7
  • ✗ Free tiers often sleep after inactivity

Hosted on: Render, Railway, Fly.io

Serverless Functions

Code runs on-demand per request. No server to manage — the platform handles everything.

Pros

  • ✓ Free/cheap at low traffic
  • ✓ Zero server management
  • ✓ Scales automatically

Cons

  • ✗ Execution time limits (10–60s)
  • ✗ No persistent connections
  • ✗ Stateless between requests

Hosted on: Vercel, Netlify, Cloudflare Workers

CORS

CORS: Connecting the Pieces

When Frontend and Backend Live on Different Domains

The Problem

When your frontend (myapp.vercel.app) makes a fetch() to your API (myapi.render.com), the browser blocks it by default.

This is CORS — Cross-Origin Resource Sharing.

The Fix

Your API must explicitly allow requests from your frontend's domain by setting the right response headers.

Express CORS Setup

// Install: npm install cors
import cors from 'cors';

app.use(cors({
  origin: 'https://myapp.vercel.app'
}));

// Or allow all origins (dev only!)
app.use(cors());

Remember from Week 4

We covered fetch() and HTTP requests. CORS is the security policy that governs cross-origin fetch() calls.

Web Application Architecture

  • ✓ Frontend SPA = static files on a CDN (fast, cheap, global)
  • ✓ API layer = server or serverless functions (compute, single region)
  • ✓ Database = managed service (Firebase, PostgreSQL)
  • ✓ Persistent servers stay running; serverless runs on-demand
  • ✓ CORS must be configured when frontend and API are on different domains
