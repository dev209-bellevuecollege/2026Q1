Simplest Path

If you're using Firebase for your backend (auth, Firestore, etc.), you just need to deploy the React frontend to Vercel. Firebase is already hosted for you.

Custom API Path

If you're building a custom Express API, deploy the frontend to Vercel and the API to Render. Remember to configure CORS!

Action Items for This Week

Deploy Early, Deploy Often

Deployment Checklist

  • ☐ Pick your hosting platform today
  • ☐ Create a free account on Vercel or Netlify
  • ☐ Deploy a "hello world" version of your project
  • ☐ Verify auto-deploy works on git push
  • ☐ If using a custom API, deploy a test endpoint to Render
  • ☐ Test that frontend can fetch() from your API
  • ☐ Share your live URL with your team

Common Mistakes to Avoid

  • ✗ Waiting until the last week to deploy
  • ✗ Forgetting to set environment variables
  • ✗ Hardcoding localhost API URLs
  • ✗ Not configuring CORS on your API
  • ✗ Committing API keys to GitHub

Pro Tip: Environment Variables

Use .env files locally and set environment variables in your hosting platform's dashboard for production API URLs and secrets.

Week 8 Recap

HTTPS, DNS & Hosting Web Applications

  • ✓ HTTPS encrypts all web traffic using TLS certificates and a chain of trust
  • ✓ DNS translates domain names to IP addresses using A and CNAME records
  • ✓ Frontend SPAs are static files hosted on CDNs (Vercel, Netlify)
  • ✓ Backend APIs run on servers (Render) or as serverless functions (Vercel)
  • ✓ All modern platforms provide auto HTTPS, GitHub integration, and free tiers
  • ✓ Deploy early — don't wait until the final week!
