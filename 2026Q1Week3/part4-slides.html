Event-Driven Programming

The Web Interaction Model

1. Register
Set up listener
→
2. Wait
User does something
→
3. Execute
Run callback

The Pattern

const button = document
  .querySelector('#myButton');

button.addEventListener(
  'click',           // event type
  function() {       // callback
    console.log('Clicked!');
  }
);

Key Insight

The callback function is not called immediately. It's stored and called later when the event occurs.

Why Callbacks?

We don't know when the user will click. The browser calls our function when it happens.

types

Common Event Types

What Users Do

Mouse Events

  • 'click' - single click
  • 'dblclick' - double click
  • 'mouseenter' - hover in
  • 'mouseleave' - hover out
  • 'mousedown' - press
  • 'mouseup' - release

Keyboard Events

  • 'keydown' - key pressed
  • 'keyup' - key released
  • 'input' - text changed

Use event.key to check which key

Form Events

  • 'submit' - form submitted
  • 'change' - value changed
  • 'focus' - element focused
  • 'blur' - lost focus

Full list: MDN Web Events Reference

e

The Event Object

Information About What Happened

Accessing Event Details

// The event object is passed to callback
button.addEventListener('click',
  function(event) {
    console.log(event.target);
    // The element that was clicked

    console.log(event.type);
    // "click"
  }
);

Useful Properties

event.target - element that triggered

event.type - event type string

event.key - which key (keyboard)

event.clientX/Y - mouse position

event.preventDefault()
Stop default browser behavior!

stop

Preventing Default Behavior

Taking Control

Form Submission Example

const form = document
  .querySelector('form');

form.addEventListener('submit',
  function(event) {
    // Stop page from reloading!
    event.preventDefault();

    // Now handle form data ourselves
    const input = document
      .querySelector('#email');
    console.log(input.value);
  }
);

Common Use Cases

Form Submit

Prevent page reload, handle with JavaScript instead

Link Click

Prevent navigation, do something else

Keyboard Input

Prevent certain characters from being typed

state

Managing Application State

Keeping Track of Things

State = Data That Changes

// State variable (outside handlers)
let count = 0;

const display = document
  .querySelector('#count');
const button = document
  .querySelector('#increment');

button.addEventListener('click',
  function() {
    // Update state
    count = count + 1;

    // Update display
    display.textContent = count;
  }
);

The Pattern

  1. Define state variables
  2. Event occurs
  3. Check/update state
  4. Update the DOM to reflect state

Key Insight: State lives outside the event handler, but is accessed inside via closure!

Debugging Events with AI

Common Problems & Solutions

Problem: Handler Not Firing

"My click handler isn't working. The button exists in HTML but nothing happens when I click."

Ask AI:

"Why isn't my click event handler firing? The element exists in HTML."

Common Causes

  • ✗ Script runs before DOM loads
  • ✗ Wrong selector (typo, wrong ID)
  • ✗ Called function instead of passing: fn() vs fn
  • ✗ Element created dynamically

Tip: Put <script> at end of <body> or use defer
