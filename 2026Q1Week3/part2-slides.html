Functions ARE Values

The Key Insight of Functional Programming

Functions Can Be Assigned

function sayHello() {
  console.log("Hello!");
}

// Assign function to variable
const greet = sayHello;

// Both call the same function
sayHello();  // "Hello!"
greet();     // "Hello!"

The Parentheses Matter!

sayHello() = execute the function

sayHello = reference the function value

Think of it like the difference between "the recipe" vs "the cooked meal"

cb

Callback Functions

Functions Passed as Arguments

What's a Callback?

A function passed to another function, to be "called back" later.

function doTwice(action) {
  action();  // call it once
  action();  // call it again
}

function sayHi() {
  console.log("Hi!");
}

doTwice(sayHi);
// "Hi!"
// "Hi!"

Anonymous Callbacks

// Define the function inline
doTwice(function() {
  console.log("Hello!");
});

// Very common pattern in JS!
// No need to name functions
// you only use once

Real Example: setTimeout(callback, 1000)

Closures

Functions Remember Their Birth Environment

The Magic of Closures

function makeGreeter(greeting) {
  // Returns a NEW function
  return function(name) {
    console.log(greeting + " " + name);
  };
}

const sayHello = makeGreeter("Hello");
const sayHola = makeGreeter("Hola");

sayHello("Alice");  // "Hello Alice"
sayHola("Bob");    // "Hola Bob"

How It Works

The inner function "closes over" the greeting variable, remembering its value even after makeGreeter has finished executing.

Key Insight: Each call creates a NEW function with its own "memory"

Closures enable: private variables, function factories, and event handlers with state

map

Array.map()

Transform Every Element

[1, 2, 3, 4]
→
map(n => n * 2)
→
[2, 4, 6, 8]

The Pattern

const numbers = [1, 2, 3, 4];

const doubled = numbers.map(
  function(n) {
    return n * 2;
  }
);

// doubled = [2, 4, 6, 8]

Real-World Example

const users = [
  { name: "Alice", age: 25 },
  { name: "Bob", age: 30 }
];

const names = users.map(
  function(user) {
    return user.name;
  }
);
// ["Alice", "Bob"]

Key: map() always returns a NEW array of the SAME length

filter

Array.filter()

Keep Only What Passes the Test

[1, 2, 3, 4, 5, 6]
→
filter(n => n % 2 === 0)
→
[2, 4, 6]

The Pattern

const numbers = [1, 2, 3, 4, 5, 6];

const evens = numbers.filter(
  function(n) {
    return n % 2 === 0;
  }
);

// evens = [2, 4, 6]

Real-World Example

const users = [
  { name: "Alice", age: 25 },
  { name: "Bob", age: 17 },
  { name: "Carol", age: 30 }
];

const adults = users.filter(
  function(user) {
    return user.age >= 18;
  }
);
// [{name: "Alice"...}, {name: "Carol"...}]

Key: Callback must return true or false

∑

Array.reduce()

Combine All Elements Into One Value

[1, 2, 3, 4]
→
reduce((acc, n) => acc + n, 0)
→
10

The Pattern

const numbers = [1, 2, 3, 4];

const sum = numbers.reduce(
  function(accumulator, n) {
    return accumulator + n;
  },
  0  // initial value
);

// sum = 10

Step by Step

Start: acc = 0

+ 1 → acc = 1

+ 2 → acc = 3

+ 3 → acc = 6

+ 4 → acc = 10

Uses: Sum, count, max/min, grouping, flattening...

→→

Method Chaining

Combine Operations Elegantly

The Power of Chaining

const users = [
  { name: "Alice", age: 25, active: true },
  { name: "Bob", age: 17, active: true },
  { name: "Carol", age: 30, active: false }
];

// Get names of active adults
const result = users
  .filter(function(u) { return u.active; })
  .filter(function(u) { return u.age >= 18; })
  .map(function(u) { return u.name; });

// result = ["Alice"]

AI Prompt Example

"Refactor this for loop to use map and filter: get the uppercase names of all users over 21"

Think in pipelines:
Data → Filter → Transform → Result

Each method returns a new array, so we can chain the next operation.
