Use Flexbox When...

Perfect for These Scenarios

  • ✓ Navigation bars — items in a row with spacing
  • ✓ Button groups — horizontal or vertical stacks
  • ✓ Centering content — both vertically and horizontally
  • ✓ Unknown number of items — dynamic content that wraps
  • ✓ Content should dictate size — items grow/shrink naturally
  • ✓ Simple card layouts — content within a single card

💡 Think: "I need to arrange items in a LINE (row or column)"

✓

Use Grid When...

Perfect for These Scenarios

  • ✓ Page layouts — header, sidebar, main, footer
  • ✓ Card grids — gallery of equal-sized items
  • ✓ Overlapping elements — items that share grid cells
  • ✓ Precise placement — exact row/column positioning
  • ✓ Layout dictates size — consistent grid structure
  • ✓ Complex magazine layouts — art-direction style designs

💡 Think: "I need a 2D GRID structure with rows AND columns"

✨

The "Magic" Grid Line

Responsive Without Media Queries

grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));

What it does:

  • • auto-fit — creates as many columns as will fit
  • • minmax(200px, 1fr) — each column is at least 200px, but can grow
  • • Result — items wrap automatically!

Why it's "magic":

  • • No media queries needed
  • • Adapts to any screen size
  • • Items maintain minimum width

💡 This single line creates a fully responsive grid layout!

Quick Decision Guide

Do you need rows AND columns?
↓ YES
Use CSS Grid
↓ NO
Use Flexbox
  • ✓ Flexbox = 1D (row OR column), content-driven
  • ✓ Grid = 2D (rows AND columns), layout-driven
  • ✓ You can use both together in the same project!
