The 3 Basic Selectors

The Foundation of CSS Targeting

💡 These three selectors are the building blocks of all CSS styling!

📄

Type Selector

Select by HTML Tag Name

What it does

Selects all elements with the specified tag name

Syntax

elementname { property: value; }

Example

p { color: purple; }
/* All paragraphs will be purple */

💡 Type selectors are broad — they affect every matching element on the page!

🔵

Class Selector

Select by Class Attribute

What it does

Selects all elements with a specific class attribute. Multiple elements can share the same class.

Syntax

.classname { property: value; }

Example

.highlighted { background-color: yellow; }
/* <p class="highlighted"> gets yellow background */

💡 Classes are reusable — the most common way to style elements!

🔴

ID Selector

Select a Unique Element

What it does

Selects a single unique element by its ID. Each ID should only appear once per page.

Syntax

#idname { property: value; }

Example

#sidebar { background-color: lightgray; }
/* <aside id="sidebar"> gets gray background */

⚠️ IDs are less flexible than classes — use sparingly for unique elements only!

Grouping Selector

One Rule, Multiple Targets

What it does

Applies the same styles to multiple selectors using commas

Syntax

selector1, selector2, selector3 { property: value; }

Example

h1, h2, h3 { font-family: Helvetica; }
/* All three heading levels get the same font */

💡 DRY Principle: Don't Repeat Yourself — group similar styles together!

💲

Compound Selector

Combining for Greater Specificity

What it does

Combines selectors without spaces — element must match ALL parts

Syntax

element.class { }   or   .class1.class2 { }

Example

p.alert { color: red; }
/* Only <p> elements with class "alert" — not <div class="alert"> */

💡 No space = AND logic. The element must be a <p> AND have class "alert"

🛒

Descendant Selector

Targeting Nested Elements

What it does

Selects elements nested inside other elements using a space

Descendant (space)

header p { color: blue; }
/* Any <p> inside <header> */

Child (>)

body > p { color: red; }
/* Only direct children */

💡 Space = any descendant. > = direct child only (one level deep)

👉

Pseudo-classes

Select by State or Position

What it does

Selects elements based on their state or position in the DOM

State-based

a:hover { color: red; }
input:focus { border: blue; }
button:active { scale: 0.95; }

Position-based

li:first-of-type { ... }
li:last-child { ... }
li:nth-of-type(even) { ... }

Example

li:nth-of-type(even) { background-color: #eee; }
/* Zebra stripes on even list items */

Selector Specificity

How the Browser Decides Which Rule Wins

The Specificity Hierarchy

ID > Class > Type

When multiple rules target the same element, higher specificity wins

💡 Think of it like a scoring system: ID = 100 points, Class = 10 points, Type = 1 point
