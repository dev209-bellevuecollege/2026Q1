Why Is This Safe?

  1. textContent treats everything as plain text — HTML tags are displayed literally, not executed
  2. document.createElement() builds DOM nodes safely without parsing HTML strings
  3. Even if someone enters <script>alert('xss')</script>, it shows up as text on the page

Vulnerable (innerHTML)

el.innerHTML = `Results for: ${query}`

Parses HTML — script tags and event handlers execute!

Safe (textContent)

el.textContent = `Results for: ${query}`

Escapes HTML — everything renders as plain text.

// THE SAFE CODE:
const query = searchInput.value;

// SAFE: Build DOM elements programmatically
const h3 = document.createElement('h3');
h3.textContent = `Showing results for: ${query}`;

// textContent escapes all HTML characters automatically.
// <script> becomes &lt;script&gt; — displayed, not executed.
